- [2026-06-14] LINKS/SLUG: el slug real del servicio es `reparacion-cortocircuitos` (una sola palabra, sin "s" intermedia). NUNCA enlazar a `/servicios/reparacion-cortos-circuitos/` (404). Por qué: 18 enlaces en 14 páginas de servicio apuntaban al slug erróneo (fuga de plantilla); si una página nueva se genera copiando otra, hereda el typo. Verificación: `grep -rl "reparacion-cortos-circuitos" --include="*.html" .` debe dar vacío. Severidad: alta.
- [2026-06-14] OPERACIÓN-PIPELINE (publicación): (1) el hook `.git/hooks/pre-push` ejecuta `node` (auto-indexación GSC) y node vive en `/usr/local/bin` pero NO está en el PATH del shell del pipeline → el primer `git push` aborta con "node: command not found". Solución: `PATH="/usr/local/bin:$PATH" git push origin main` (NO usar `--no-verify`: saltarse el hook omite la indexación en Google, que es parte del flujo). (2) zsh NO hace word-splitting de variables sin comillas en `for f in $files` → un `sed -i` en lote falla silenciosamente (git diff vacío). Usar `grep -rl ... | while IFS= read -r f; do ...; done` y SIEMPRE confirmar con `git diff --stat` que los archivos cambiaron. Severidad: media.
- [2026-06-16] A11Y/CSS: la clase `.sr-only` (oculto visual para lectores de pantalla) NO está definida en ningún CSS externo (styles.css / .min / .hash); SOLO vive en el `<style>` crítico inline de `index.html`. Si añades markup con `class="sr-only"` (p.ej. `<label>` de formulario) a OTRA página, DEBES añadir también la regla `.sr-only{position:absolute;width:1px;height:1px;padding:0;margin:-1px;overflow:hidden;clip:rect(0,0,0,0);white-space:nowrap;border:0}` al `<style>` inline de ESA página, o el texto se renderiza VISIBLE (regresión). Por qué: los formularios de 3 páginas (1 servicio + 2 blog) habían perdido sus `<label>` y al reponerlos sin la definición habrían quedado visibles. Verificación: por página, `grep -c '\.sr-only{'` debe ser ≥1 si hay `class="sr-only"`. Severidad: media.
- [2026-06-16] PIPELINE/GSC: el subagente `revisor-gsc` está cableado a `mcp__local-seo__*` (apunta a `sc-domain:plomeroculiacanpro.mx`, OTRO sitio) y SIEMPRE emite "verificación ciega" para electricista. PERO el servidor MCP `gsc` del entorno SÍ tiene la propiedad `https://electricistaculiacanpro.mx/` (verificado con `mcp__gsc__gsc_list_sites`). O sea: la indexación NO está ciega; el revisor está mal configurado. Acción humana pendiente: reapuntar el agente `revisor-gsc` a las herramientas `mcp__gsc__*` con esa propiedad. Severidad: media (no es ceguera real, es un falso positivo del revisor).
- [2026-06-17] SEO/SOCIAL (leak de plantilla): al corregir un canonical/og:url filtrado de la plantilla origen, REVISA TAMBIÉN `twitter:url` — es un tercer campo de URL que se filtra por separado. El 2026-06-14 se arreglaron canonical+og:url de 4 páginas de servicio (apuntaban a `/servicios/reparacion-cortocircuitos/`) pero el `twitter:url` de esas MISMAS 4 (cambio-cableado-electrico, electricista-a-domicilio, instalacion-centro-carga, instalacion-ventiladores-techo) quedó apuntando a cortocircuitos y sobrevivió 3 días. Verificación site-wide: por cada HTML, `rel="canonical"` y `twitter:url` (normalizando trailing slash) deben COINCIDIR; cualquier mismatch = leak. La home y los servicios legítimos ya coinciden. Severidad: media. Arreglado 2026-06-17.
